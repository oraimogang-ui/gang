Blueprint • Conceptual

Project “Chameleon”

End-to-end automation framework for a modern animation studio, orchestrating every phase from script ingestion to final distribution with data-driven visibility.

Version

2.0 • Conceptual Blueprint

Objectives

Increase efficiency, reduce errors, fortify security, and deliver real-time production insight.

Guiding Tone

Technical, authoritative, and professional—crafted for pipeline and production stakeholders.

Core Principles

Single Source of Truth
  • All assets, tasks, and statuses flow through the production tracker (ShotGrid, Ftrack).
  • Automations pull inputs and write results directly to the tracker.
API-First Integration
  • Every pipeline tool exposes APIs for automation and data exchange.
  • Enables consistent validation, publishing, and notifications.
Automate, Don’t Annihilate
  • Targets repetitive, non-creative tasks while empowering artist choices.
  • Human approvals and creative reviews remain explicit milestones.
Data-Driven Decisions
  • Continuous logging feeds dashboards for progress, budget, and capacity.
  • Alerts highlight bottlenecks before they threaten delivery.
Scalability
  • On-demand compute through cloud bursting augments the render farm.
  • Elastic storage tiers balance high-performance NAS with archival.

Pipeline Phases & Automation

Phase 1: Pre-ProductionIngestion
  • Script Ingestion: Parse commits, tag scenes/characters/locations, and seed tracker tasks for storyboard, VisDev, and modeling.
  • Asset Publish: Validate naming and tech checks, version assets in DAM, set status to Pending Review, and notify supervisors.
Phase 2: ProductionShots
  • Scene Assembly: Generate base scenes, pull approved assets, save to project, and link back to the tracker.
  • Dailies: Render playblasts with burn-ins, transcode to MP4, upload to review systems, and add to playlists.
Phase 3: Post-ProductionRender & Comp
  • Render Farm: Prioritize and submit jobs; burst to cloud when local queues spike.
  • Post-QC: Scan sequences for missing frames, update to Ready for Comp, and pre-build compositing scripts with AOVs.
Phase 4: Editorial & SoundConform
  • Edit Conform: Parse new EDL/XML, flag new or retimed shots, and adjust tracker statuses for downstream work.
Phase 5: FinalizationDelivery
  • Mastering: Transcode locked episodes to required profiles (4K ProRes, HD H.264, platform variants).
  • Packaging & Delivery: Bundle deliverables, create IMF/DPP packages, deliver via high-speed transfer, and log checksums.

Core Technology Stack

Tracking: Autodesk ShotGrid
DAM: Integrated with ShotGrid / Alienbrain
DCCs: Maya, Houdini, Substance Suite, Nuke
Version Control: Perforce (binaries) + Git (scripts)
Render Management: AWS Thinkbox Deadline
Compute: On-prem + AWS EC2 Spot
Storage: NAS + S3 Glacier
Scripting: Python 3, ShotGrid API, PySide/PyQt
Communication: Slack bots, Email
Review: ShotGrid RV, SyncSketch
Built to keep automation authoritative, observable, and artist-first.
